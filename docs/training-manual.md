# NocoCRM 培训手册（T17.04）

面向 Better Bags 各岗位的操作手册。配合 `training-storyboard.md` 录屏使用。

> 目标：新员工照本手册可独立完成首单全流程。

---

## 第 1 章 业务员日常操作

### 1.1 接收询盘 → 创建客户
1. 客户管理 → 客户 → 新建
2. 填公司名、国家、来源、主要联系人、邮箱（必填）
3. 保存后 `customer_code` 自动生成（CUST-YYYYMM-xxx），系统按来源/年单量自动建议 A/B/C/D 等级
4. 工作时间内系统会在 4h 内给你创建跟进待办

### 1.2 录入询盘 → 完整度评分
1. 询盘报价 → 询盘 → 新建，关联客户
2. 尽量填全：产品分类、数量、材料、颜色、设计稿、目标价、期望交期
3. 保存后看 `completeness_score`：
   - < 80 → 状态自动"需求完善"，系统提醒你补全
   - ≥ 80 → 状态自动"已交核价"，通知核价员

### 1.3 报价（自动化最强的一步）
1. 等核价员出 `cost_estimates`（你只能看到建议价 final_price，看不到成本）
2. 报价单 → 新建，关联询盘
3. 填单价、数量后**留空工厂和贸易条款**：
   - 数量 ≥ 1500 且非急单 → 系统自动路由缅甸仰光 + FOB Yangon
   - 否则 → 山东日照 + FOB Qingdao
4. 付款条款：新客户只能选 T/T 30/70（系统强制）
5. 交期：缅甸大货系统强制 ≥ 90 天
6. 提交时如果单价低于建议价，系统自动锁回草稿等经理审批

### 1.4 打样
1. 报价确认后 → 样品 → 新建，关联报价
2. 委托日期填上，计划完成自动 = +7 天，打样工厂默认青岛
3. 客户驳回时 revision_no +1；**到第 4 次系统自动把客户标记高风险**，经理会介入

### 1.5 跟进话术
- 报价 14 天没决策 → 系统提醒你跟进
- 跟进话术 表里有 4 套模板（决策询问/新案例/技术补充/节日问候）× 中英日，直接复制用

## 第 2 章 销售经理

### 2.1 报价审批
- 业务员报价低于成本线时，报价单 `pending_approval_reason` 会显示原因，状态停在草稿
- 你评估后手动改回"已发送"

### 2.2 谈判止损
- 谈判记录 ≥ 5 轮，系统在报价单写入"建议止损评估"
- 你判断是否继续

### 2.3 看板与报表
- 客户 Kanban 按 status 分列，A 类置顶
- 报表（M16）：13 阶段漏斗 / 销售业绩 / 打样转化率，识别瓶颈

## 第 3 章 财务

### 3.1 定金确认（触发排产）
1. 合同签署后系统自动建订单 + 30% 定金 payment
2. 收到定金后，把该 payment 状态改 `已结清`
3. **系统自动**：订单解锁到"待生产" + 创建 70% 尾款 payment + 创建排产记录

### 3.2 尾款催收
- QC 通过后订单变"已完成"，系统提示发尾款邀请邮件
- 收到尾款标记 `已结清` → 订单 balance_paid=true（发货前提）

### 3.3 现金流报表
- `payments:cashFlow` action：本月应收定金/尾款、已收、逾期一目了然

### 3.4 字段隔离说明
- 你看不到客户的电话/邮箱/微信（隐私保护），但能看所有金额字段

## 第 4 章 工厂排产与质检

### 4.1 物料齐套时间轴
- 物料 表录入采购下单日，系统自动算：
  - 到东莞 = +14 天
  - 验货通过 = +21 天
  - 到工厂 = +35 天
- 物料逾期系统告警采购

### 4.2 排产
- 定金到账系统自动建排产记录（待开工）
- 排产甘特图查看 planned vs actual
- 5 个里程碑：裁断 → 缝制 → 钉胶 → 检针(X线) → 包装

### 4.3 质检
- 质检报告 AQL 默认 2.5
- 勾选"最终内检" + "通过" → 订单自动变"已完成"，触发尾款邀请
- 生产异常填到生产进度的 issues，系统通知业务员

### 4.4 发货文件
- 切换发货状态到"已开船"前，必须上传发票/装箱单/原产地证（系统强制校验）

## 第 5 章 管理员

### 5.1 角色分配（9 个角色）
- 用户管理 → 分配角色：admin / sales_manager / sales / finance / factory_qd / factory_ygn / qc / procurement / readonly
- 数据隔离自动生效（业务员只看自己客户，工厂只看自己工厂订单）

### 5.2 插件启用顺序
core → meetings → quotation → followup → orders → fulfillment → retention → acl → reports → ops

### 5.3 备份还原
- `bash deploy/backup.sh` 手动备份
- crontab 每天 2 点自动备份
- 还原：`yarn nocobase restore <file>`

### 5.4 监控告警
- 配置 OPS_ALERT_EMAIL / OPS_ALERT_WEBHOOK
- 关键工作流加失败分支调用 ops:sendAlert
