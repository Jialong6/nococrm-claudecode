# M13 + M14 端到端测试清单 (T13.06 + T14.05)

## 准备
- 启用 plugin-nococrm-core + quotation + orders + fulfillment + retention
- 启用 plugin-workflow* / plugin-notification-email / plugin-public-forms
- 至少 1 个 status='shipping_pending' 或更后的订单（含 shipment）
- 1 个 status='new' 的新客户 + 1 个 status='closed' 已成交客户

---

## 场景 1：delivery_feedback 自动同步（T13.01）
1. 创建 delivery_feedback，只填 shipment + satisfaction_score=5
- 期望：
  - [ ] order 自动 = shipment.order
  - [ ] customer 自动 = order.customer

## 场景 2：低分反馈触发补救（T13.04）
1. 创建 delivery_feedback：satisfaction_score=2, quality_issues="缝线有瑕疵"
- 期望：
  - [ ] customer.risk_flag = true
  - [ ] order.notes 追加 `[YYYY-MM-DD LOW_SATISFACTION] 客户评分 2/5...`

## 场景 3：高分反馈清除风险（T13.04）
1. 找一个 risk_flag=true 的客户
2. 该客户新订单收到 5 分反馈
- 期望：
  - [ ] customer.risk_flag = false

## 场景 4：3 分反馈中性不触发
1. satisfaction_score=3
- 期望：
  - [ ] customer.risk_flag 保持原值（既不标也不清）

## 场景 5：首单完成 → customer.status='closed'（T13.05）
1. 找一个 status='new' 客户的首单
2. order.status 改为 'delivered'
3. order.balance_paid 改为 true（如果之前没付）
4. 保存
- 期望：
  - [ ] customer.status = 'closed'
  - [ ] customer.last_order_at = 当前时间

## 场景 6：多单完成 → customer.status='maintaining'（T13.05）
1. 同一客户的第二个订单 status='delivered' + balance_paid=true
- 期望：
  - [ ] customer.status = 'maintaining'
  - [ ] last_order_at 更新

## 场景 7：long_term_followups 创建
1. 手动给某客户创建 long_term_followup，type=quarterly_review
- 期望：
  - [ ] 字段齐全（含 outcome 默认 pending）

## 场景 8：长期客户新询盘回流（T14.04）
1. 找一个 status='maintaining' 客户
2. 给该客户创建一个新 inquiry
- 期望：
  - [ ] customer.status 切回 'following'
  - [ ] customer.last_contact_at = 当前时间
  - [ ] customer.notes 追加 `[YYYY-MM-DD LONG_TERM_REQUEST] 长期维护客户产生新询盘...`

## 场景 9：14 天感谢信定时（T13.03）
1. 配置 README 中的定时工作流
2. 找一个 14 天前 delivered 的 shipment（mock delivered_at）
3. 该 shipment 还没有 delivery_feedback
4. 手动触发或等定时
- 期望：
  - [ ] customer.email 收到 thank-you-survey 邮件
  - [ ] 邮件中 survey_url 可点击打开公共反馈表单

## 场景 10：内部复盘记录（T13.02）
1. 给某 order 创建 internal_retro
2. participants 选 3 个 user（业务员 + QC + 工厂）
3. 填 what_went_well / what_went_wrong / improvement_actions
- 期望：
  - [ ] 记录保存成功
  - [ ] participants 列表显示 3 人

## 场景 11：180 天不下单降级（T14.03）
1. 配置 README 中的每日定时
2. 找一个 priority='A' 客户，把 last_order_at 改为 200 天前
3. 触发定时
- 期望：
  - [ ] priority 改为 'B'
  - [ ] notes 追加降级原因
